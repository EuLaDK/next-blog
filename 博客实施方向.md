# 博客项目实施方向

> 更新时间：2026-06-22

## 当前方向

项目先做高完成度前台原型，后端、登录、后台管理和真实 Markdown 渲染暂时后置。

前台采用“杂志策展 + 时间轨道归档”的组合：

- 首页和 `/home` 保持杂志式文章浏览体验，强调文章筛选、搜索、精选内容和阅读效率。
- `/home?view=archive` 使用时间轨道归档视图，按年份和月份组织文章，适合查看写作脉络。
- `/posts/[slug]` 先保留文章阅读预览，后续再接真实内容源。
- `/plan` 作为项目路线静态页面，后续只补功能，不再把静态视觉细节写进本文档。
- 数据层先用 `src/lib/blog-data.ts` 的 typed mock data，字段尽量贴近未来 Prisma 或 Markdown 内容模型。

## 当前代码边界

- `src/app/**/page.tsx`：页面入口，负责读路由参数、准备数据、组合页面组件。
- `src/components/header/Header.tsx`：全局导航、用户菜单、移动端菜单。
- `src/components/header/navigation.ts`：导航配置和 active 判断纯函数。
- `src/components/home/BlogExperience.tsx`：首页体验总控，负责搜索、分类、标签等交互状态。
- `src/components/home/LeftMenu.tsx`：左侧分类和标签筛选，只通过 props 与 `BlogExperience` 通信。
- `src/components/home/RighTools.tsx`：右侧搜索、热门文章、阅读工具入口。
- `src/components/home/MainContent.tsx`：首页中心内容和文章流。
- `src/components/home/ArchiveTimeline.tsx`：时间轨道归档视图。
- `src/components/post/PostPreview.tsx`：文章详情预览页。
- `src/lib/blog-data.ts`：当前 mock 数据、类型定义和纯函数。

原则：页面负责拿数据，`BlogExperience` 负责交互状态，子组件只接收 props 展示或触发回调。

## 推荐开发节奏

建议每个小功能一个 commit：

1. `feat: improve header navigation`
2. `feat: sync home filters with url`
3. `feat: add shared filter summary`
4. `feat: render post sections from data`
5. `feat: add reading progress`
6. `feat: scaffold studio page`

每次提交前做限定范围验证：

```bash
npm.cmd run lint -- src/components/header src/app/home src/components/home
node --disable-warning=MODULE_TYPELESS_PACKAGE_JSON --test src/lib/blog-data.test.ts
node --disable-warning=MODULE_TYPELESS_PACKAGE_JSON --test src/components/header/navigation.test.ts
```

如果只改了某一个模块，就只跑对应模块的 lint 和测试。

### 下一步 1：让文章详情页也接 Prisma

目标：`/posts/[slug]` 不再从 `allPosts` 找文章，而是根据 slug 查询数据库。

涉及文件：

- 修改：`src/lib/posts.ts`
- 修改：`src/app/posts/[slug]/page.tsx`

建议实现：

- 在 `src/lib/posts.ts` 里新增 `getPostDetailBySlug(slug: string)`。
- 这个函数内部用 `prisma.post.findUnique` 查询文章，并 include `author`、`category`、`tags.tag`。
- 查不到文章时返回 `null`，由页面调用 `notFound()`。
- 详情页继续把 Prisma 数据交给 `mapPrismaPostToBlogPost` 转成前台 `Post`。

模块交互：

- `posts/[slug]/page.tsx` 只负责拿 slug 和决定是否 404。
- `src/lib/posts.ts` 负责数据库查询和数据转换。
- `PostPreview.tsx` 不关心数据来自 mock 还是 Prisma，只接收 `Post`。

验收：

```bash
npm.cmd run lint -- src/lib/posts.ts src/app/posts/[slug]/page.tsx
npx.cmd tsc --noEmit --pretty false --project tsconfig.json
```

### 下一步 3：把筛选状态真正同步回 URL

目标：用户点击分类、标签、搜索时，URL 跟着变化，刷新后状态还能恢复。

涉及文件：

- 修改：`src/components/home/BlogExperience.tsx`

建议实现：

- 使用 `useRouter`、`usePathname`、`useSearchParams`。
- 封装一个 `replaceQuery(nextState)` 小函数，用 `router.replace` 更新 URL。
- 点击分类时写入 `category`，点击标签时写入 `tag`，搜索时写入 `q`。
- 清空筛选时删除 `category`、`tag`、`q`，保留 `view=archive` 这种视图参数。

注意：

- 这一步会触发 App Router 重新渲染当前 route，但前台状态会和 URL 更一致。
- 如果担心每次输入搜索都触发，可以先只在回车或防抖后更新 URL。

验收：

- `/home?category=nextjs` 刷新后仍然选中 Next.js 分类。
- 在归档页筛选后，URL 保留 `view=archive`。
- 清空筛选后 URL 不留下空参数。

### 下一步 4：整理内容编辑方案

目标：先决定文章正文最终怎么存。

推荐路线：

- 短期：数据库 `Post.content` 存 Markdown 字符串。
- 前台：详情页先把 Markdown 按标题拆成 sections，后续再接 Markdown 渲染器。
- 后台：以后做 `/studio` 时再考虑富文本编辑器，先不要一上来做复杂编辑器。

原因：

- 博客内容更适合 Markdown，方便版本管理和迁移。
- 富文本编辑器后面可以作为后台体验增强，不影响当前 Prisma 内容层。
