# 博客项目实施方向

> 更新时间：2026-06-18

## 当前方向

项目先做高完成度前台原型，后端、登录、后台管理和真实 Markdown 渲染暂时后置。

前台采用“杂志策展 + 时间轨道归档”的组合：

- 首页和 `/home` 保持杂志式文章浏览体验，强调文章筛选、搜索、精选内容和阅读效率。
- `/home?view=archive` 使用时间轨道归档视图，按年份和月份组织文章，适合查看写作脉络。
- 文章详情先做阅读预览页原型，后续再接真实内容源。
- 数据层先用 `src/lib/blog-data.ts` 的 typed mock data，字段尽量贴近未来 Prisma 模型，方便后续替换。

## 已完成内容

### 前台结构

- 完成公共博客首页和 `/home` 的杂志式三栏布局。
- 完成 Header 导航、左侧分类标签栏、中央文章流、右侧搜索和工具栏。
- 修复文章导航和归档导航的 active 状态混淆问题。
- 保留移动端响应式基础，桌面为左中右三栏，窄屏收敛为更适合阅读的布局。

### 数据和筛选

- 建立 `Author`、`Category`、`Tag`、`Post`、`Announcement` 等 mock 数据结构。
- 完成本地搜索、分类筛选、标签筛选。
- 完成文章精选、热门文章、相关文章、上一篇/下一篇等纯函数。
- 新增 `getArchiveGroups`，用于按年份和月份生成归档分组。

### 文章预览

- 完成 `/posts/[slug]` 的文章详情/阅读预览页原型。
- 支持文章元信息、封面视觉、正文预览、相关文章、上一篇/下一篇。
- 目前仍使用 mock 内容，不接真实 Markdown 文件或数据库。

### 时间轨道归档

- 新增 `ArchiveTimeline` 组件。
- `/home?view=archive` 已接入时间轨道视图。
- 归档视图支持当前搜索、分类、标签筛选后的结果展示。
- 提供筛选后无结果的设计化空状态和重置按钮。

### 验证记录

- `src/lib/blog-data.test.ts` 数据层测试通过。
- 限定范围 lint 通过：`src/app/home`、`src/components/home`、`src/lib`。
- 本地服务验证：
  - `/home` 返回普通文章浏览内容。
  - `/home?view=archive` 返回时间轨道内容。

## 下一步建议

### 优先级 1：把归档入口做完整

- 将 Header 的“归档”入口明确指向 `/home?view=archive`。
- 检查 Header active 状态，确保文章、归档、用户页互不影响。
- 在移动端菜单中同步加入归档入口。

### 优先级 2：做一次前台视觉验收

- 检查桌面、平板、手机宽度下的首页、归档页、文章预览页。
- 修正文字溢出、卡片拥挤、按钮换行、右侧工具栏压缩等问题。
- 优化时间轨道在移动端的节奏，避免年份块和文章卡片占用过多空间。

### 优先级 3：补齐前台交互细节

- 搜索输入增加更明确的清空状态。
- 分类和标签筛选后的状态提示再统一一下。
- 主题切换如果要保留，需要决定是视觉演示还是本地持久化。
- 文章卡片点击、hover、空状态、加载更多按钮保持一致的交互风格。

### 优先级 4：准备内容层方案

- 决定文章来源：
  - 方案 A：本地 Markdown/MDX 文件，适合个人博客早期快速维护。
  - 方案 B：数据库 + Prisma，适合后续后台管理、草稿、发布流。
  - 方案 C：混合方案，公开文章来自 Markdown，统计、标签、用户行为来自数据库。
- 决定代码高亮、目录、阅读进度、文章摘要等阅读页能力。

### 优先级 5：再进入后端和后台

- Prisma schema 和数据库表设计。
- NextAuth 登录。
- 后台文章 CRUD。
- Markdown 上传或编辑器。
- 阅读数、搜索、草稿、发布状态、归档页服务端查询。

## 暂不做的内容

- 不急着接数据库。
- 不急着做后台管理。
- 不急着做真实 Markdown 渲染。
- 不急着做图片上传。
- 不急着做部署、Docker、CI/CD。

当前阶段最重要的是把前台体验打磨成一个能真实浏览、能展示方向、后续好接数据的高完成度原型。
