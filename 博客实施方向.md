# 博客项目实施方向

> 更新时间：2026-06-22

## 当前方向

项目先做高完成度前台原型，后端、登录、后台管理和真实 Markdown 渲染暂时后置。

前台采用“杂志策展 + 时间轨道归档”的组合：

- 首页和 `/home` 保持杂志式文章浏览体验，强调文章筛选、搜索、精选内容和阅读效率。
- `/home?view=archive` 使用时间轨道归档视图，按年份和月份组织文章，适合查看写作脉络。
- `/posts/[slug]` 先保留文章阅读预览，后续再接真实内容源。
- `/plan` 作为项目路线静态页面，后续只补功能，不再把静态视觉细节写进本文档。
- 数据层先用 `src/lib/blog-data.ts` 的 typed mock data，字段尽量贴近未来 Prisma 或 Markdown 内容模型。

## 当前代码边界

- `src/app/**/page.tsx`：页面入口，负责读路由参数、准备数据、组合页面组件。
- `src/components/header/Header.tsx`：全局导航、用户菜单、移动端菜单。
- `src/components/header/navigation.ts`：导航配置和 active 判断纯函数。
- `src/components/home/BlogExperience.tsx`：首页体验总控，负责搜索、分类、标签等交互状态。
- `src/components/home/LeftMenu.tsx`：左侧分类和标签筛选，只通过 props 与 `BlogExperience` 通信。
- `src/components/home/RighTools.tsx`：右侧搜索、热门文章、阅读工具入口。
- `src/components/home/MainContent.tsx`：首页中心内容和文章流。
- `src/components/home/ArchiveTimeline.tsx`：时间轨道归档视图。
- `src/components/post/PostPreview.tsx`：文章详情预览页。
- `src/lib/blog-data.ts`：当前 mock 数据、类型定义和纯函数。

原则：页面负责拿数据，`BlogExperience` 负责交互状态，子组件只接收 props 展示或触发回调。

## 后续功能计划

### 阶段 4：文章详情改成按文章数据渲染

现在 `PostPreview.tsx` 里的正文段落是固定 mock。下一步要让每篇文章拥有自己的正文结构。

#### 1. 扩展文章数据类型

涉及文件：

- 修改：`src/lib/blog-data.ts`
- 修改：`src/lib/blog-data.test.ts`

新增数据：

- `PostSection`
- `Post.sections`

测试建议：

- 每篇文章至少有一个 section。
- 每个 section 都有 title 和 body。

#### 2. PostPreview 使用 post.sections

涉及文件：

- 修改：`src/components/post/PostPreview.tsx`

模块交互：

- `/posts/[slug]/page.tsx` 仍通过 `getPostBySlug` 拿文章。
- `PostPreview` 从 `post.sections` 渲染正文。
- 阅读目录从同一份 `post.sections` 生成。

验收：

- 不同文章详情页显示不同正文。
- 目录标题跟正文标题一致。
- 上一篇、下一篇、相关推荐不受影响。

### 阶段 5：阅读页增强

#### 1. 阅读进度条

目标：文章详情页滚动时显示阅读进度。

涉及文件：

- 新增：`src/components/post/ReadingProgress.tsx`
- 修改：`src/components/post/PostPreview.tsx`

模块交互：

- `ReadingProgress` 是 Client Component。
- `PostPreview` 可以继续保持主要展示职责，只嵌入 `ReadingProgress`。

验收：

- 文章页向下滚动时进度增加。
- 离开文章页不影响首页。

#### 2. 目录当前章节高亮

目标：阅读文章时，右侧目录能高亮当前章节。

涉及文件：

- 新增：`src/components/post/PostToc.tsx`
- 修改：`src/components/post/PostPreview.tsx`

模块交互：

- `PostPreview` 给正文 section 设置稳定 id。
- `PostToc` 接收目录数据和 section id。
- `PostToc` 自己管理当前 active section。

验收：

- 点击目录能跳转到对应段落。
- 滚动正文时目录高亮变化。

### 阶段 6：内容层先接 Markdown

建议先使用本地 Markdown，不急着上 Prisma。

推荐目录：

```txt
content/posts/
  nextjs-cache-field-notes.md
  markdown-editor-studio.md
```

#### 1. 建立 posts 读取模块

涉及文件：

- 新增：`content/posts/*.md`
- 新增：`src/lib/posts.ts`
- 逐步调整：`src/lib/blog-data.ts`

模块职责：

- `src/lib/posts.ts` 读取 Markdown 文件。
- 解析 frontmatter。
- 返回接近 `Post` 的结构。

第一步先读取：

- `slug`
- `title`
- `excerpt`
- `publishedAt`
- `category`
- `tags`
- `readingTime`

#### 2. 页面改用真实内容源

涉及文件：

- 修改：`src/app/page.tsx`
- 修改：`src/app/home/page.tsx`
- 修改：`src/app/posts/[slug]/page.tsx`

模块交互：

- 页面负责读取 Markdown。
- `BlogExperience`、`MainContent`、`ArchiveTimeline`、`PostPreview` 尽量不关心数据来自 mock 还是 Markdown。

验收：

- 新增 Markdown 文件后，首页能出现新文章。
- 文章详情页能通过 slug 找到对应 Markdown 内容。

### 阶段 7：后台之前先做工作台壳子

不要马上做完整后台。先做一个静态内容工作台，练管理端页面布局和组件拆分。

涉及文件：

- 新增：`src/app/studio/page.tsx`
- 新增：`src/components/studio/StudioShell.tsx`
- 新增：`src/components/studio/PostDraftList.tsx`

功能结构：

- 左侧：草稿、已发布、分类、标签。
- 中间：文章列表。
- 右侧：发布检查清单。

暂时不做：

- 不保存草稿。
- 不登录。
- 不接数据库。
- 不上传图片。

模块交互：

- `studio/page.tsx` 准备静态 mock draft 数据。
- `StudioShell` 负责页面结构。
- `PostDraftList` 负责显示草稿列表。

验收：

- `/studio` 页面可访问。
- 不影响博客前台页面。
- 后续可以自然接 Prisma 和 NextAuth。

## 推荐开发节奏

建议每个小功能一个 commit：

1. `feat: improve header navigation`
2. `feat: sync home filters with url`
3. `feat: add shared filter summary`
4. `feat: render post sections from data`
5. `feat: add reading progress`
6. `feat: scaffold studio page`

每次提交前做限定范围验证：

```bash
npm.cmd run lint -- src/components/header src/app/home src/components/home
node --disable-warning=MODULE_TYPELESS_PACKAGE_JSON --test src/lib/blog-data.test.ts
node --disable-warning=MODULE_TYPELESS_PACKAGE_JSON --test src/components/header/navigation.test.ts
```

如果只改了某一个模块，就只跑对应模块的 lint 和测试。

当前工作区里 `src/lib/blog-data.test.ts` 有一个未提交改动：导入路径从 `./blog-data.ts` 改成了 `./blog-data`。正式继续开发前，先确认这个改动是否要保留。如果保留，建议先跑：

```bash
node --disable-warning=MODULE_TYPELESS_PACKAGE_JSON --test src/lib/blog-data.test.ts
```

确认通过后再继续做下一项功能。
