Next.js 全栈个人博客项目（基础 + 进阶增强版）
基于 Next.js 16 + App Router + TypeScript + TailwindCSS + Prisma + NextAuth.js 技术栈，拆分基础必做功能、进阶难点功能、高阶挑战功能，同时附带详细需求、技术考点、实现提示，循序渐进拔高能力，做完可作为简历主力项目。
整体定位：前后端一体、带权限后台、支持富文本 / Markdown、SEO 优化、缓存策略、性能优化、上线部署，覆盖 Next.js 核心难点与工程化实践。
一、项目整体架构 & 前置技术约定
技术栈固定
框架：Next.js 15（纯 App Router，不混用 Pages Router）
语言：TypeScript（严格类型，禁止 any）
样式：Tailwind CSS + 自定义主题
数据库：Prisma ORM + 开发环境 SQLite / 生产 PostgreSQL
身份认证：NextAuth.js v5（Credentials 账号密码 + 可选 Github 三方登录）
内容处理：markdown-it + shiki 代码高亮 + 图片懒加载
表单 / 校验：React Hook Form + Zod 强类型校验
其他：原生 Next.js 图片 / 字体 / 元数据、API 路由、中间件、缓存策略
数据库表设计（Prisma Schema 核心表）
提前定义表结构，所有功能围绕表展开：
User：用户表（id、用户名、邮箱、密码哈希、角色 role(ADMIN/USER)、头像、创建时间）
Post：文章表（id、标题、摘要、正文、封面图、状态 status(DRAFT/PUBLISHED)、发布时间、更新时间、作者 ID、分类 ID）
Category：分类表（id、分类名、排序、简介）
Tag：标签表（id、标签名）
PostTag：文章 - 标签中间表（多对多关联）
ViewRecord：阅读记录表（统计文章访问量，防刷）
二、第一部分：基础核心功能（必做，巩固官网知识点）
目标：掌握路由、服务端 / 客户端组件、基础 CRUD、渲染模式、简单认证

全局通用布局：顶部导航栏、侧边栏（分类 / 标签）、页脚，全局 Layout 嵌套
响应式适配：PC / 平板 / 手机三端兼容，移动端汉堡菜单

全局 404 页面、500 错误页面、加载状态骨架屏
技术考点

全局错误处理、loading.tsx 加载状态
模块 2：前台用户端（访客可见）
1. 首页
展示已发布文章列表：封面图、标题、摘要、发布时间、分类、阅读量
基础分页功能（服务端分页）
顶部热门文章 / 公告位
2. 文章分类页
按分类筛选文章，URL 动态路由 /categories/[slug]
分类列表聚合展示
3. 文章标签页
按标签筛选文章，动态路由 /tags/[slug]
4. 文章详情页（核心）
Markdown 正文渲染、代码块语法高亮、行号显示
文章元信息：作者、发布时间、阅读量、所属分类 / 标签
上一篇 / 下一篇文章跳转
渲染模式要求：使用 ISR 增量静态再生，设置重验证时间
5. 简单搜索页
输入关键词，服务端模糊匹配文章标题 / 摘要，返回结果
6. 关于页面
静态个人介绍页面，纯 SSG 静态生成

1. 登录 / 注册页面
账号密码登录、新用户注册
表单校验：非空、邮箱格式、密码长度
登录状态持久化，未登录拦截后台路由
2. 后台首页 Dashboard
数据统计卡片：文章总数、草稿数、分类数、总阅读量
最近发布文章列表
3. 文章基础管理
文章列表：区分草稿 / 已发布状态
新建文章、编辑文章、删除文章（二次确认弹窗）
文章状态切换：草稿 ↔ 发布
4. 分类 / 标签管理
分类：新增、编辑、删除、排序
标签：新增、批量删除
技术考点（基础阶段）
动态路由、路由分组 (admin) 区分前后台
服务端 API 路由 route.ts 编写 CRUD 接口
NextAuth 会话管理、客户端获取登录态
Prisma 联表查询、分页、条件筛选
ISR / SSG / CSR 三种渲染模式实战
基础表单、弹窗、状态管理
三、第二部分：进阶功能（拔高难度，职场高频考点）
这部分是区分新手和熟练开发者的关键，重点考察 Next.js 高级特性、工程化、权限、缓存、安全
模块 1：认证 & 权限体系（进阶）
功能需求
路由中间件权限控制
使用 Next.js middleware.ts 全局拦截：未登录禁止访问后台、普通用户禁止进入管理页（仅 ADMIN 角色可访问）
路由白名单：登录页、前台页面放行
三方登录：接入 Github 快捷登录（NextAuth 多提供商配置）
密码安全：密码哈希加密存储（禁止明文），登录密码加盐校验
会话管理：自定义登录过期时间、退出登录、多端会话兼容
技术难点
Middleware 中间件逻辑编写、匹配路由规则
NextAuth v5 复杂配置、多 OAuth 提供商
服务端密码加密（bcrypt）、安全规范
模块 2：编辑器 & 内容增强（前端难点）
功能需求
双模式编辑器：后台支持「纯 Markdown 源码编辑」+「可视化富文本预览」双栏布局
图片上传功能
编辑器内选择本地图片，上传至服务端，返回图片链接自动插入 Markdown
限制图片大小、格式，上传加载态、错误提示
草稿自动保存
编辑文章时，前端定时本地缓存 + 服务端临时草稿保存，防止页面刷新丢失内容
文章摘要自动生成：根据正文截取前 N 字符作为摘要，支持手动修改
技术难点
客户端大表单状态管理、分栏布局
服务端文件上传 API、文件校验、存储策略
本地存储（localStorage）+ 服务端双缓存
Markdown 编辑器交互逻辑
模块 3：缓存 & 渲染策略深度实践（Next.js 核心难点）
功能需求
精细化缓存控制
首页、分类页、标签页：使用 ISR，区分公开内容缓存
文章详情页：ISR + 按需重验证，发布新文章后主动触发缓存刷新
后台管理页：纯 CSR / SSR（禁止静态缓存，保证数据实时）
按需重验证（Revalidate）
后台发布 / 编辑 / 删除文章后，调用 revalidatePath / revalidateTag 主动清除前台缓存，做到更新内容前台立刻生效
组件级缓存：合理使用 cache()、revalidate 控制服务端组件缓存
技术难点
Next.js 15 缓存机制理解、标签缓存（Cache Tags）
主动重验证 API 使用、缓存失效逻辑
区分不同页面的渲染方案，解决「静态缓存不更新」经典问题
模块 4：阅读统计 & 防刷机制（业务逻辑进阶）
功能需求
文章阅读量统计：每篇文章展示累计阅读数
防重复刷量：
基于客户端 IP + 浏览器 Cookie，同一用户 24 小时内多次访问同一篇文章，只计 1 次阅读
后台数据看板：阅读量趋势图表（使用简单图表库如 Recharts）
技术难点
API 中获取客户端真实 IP
Cookie 读写、时效控制
服务端条件判断、数据聚合统计
服务端组件渲染图表（客户端组件封装图表）
模块 5：搜索功能升级（全文字检索）
功能需求
替换基础模糊搜索，实现全文搜索（标题 + 摘要 + 正文）
搜索联想：输入关键词实时下拉联想推荐
搜索结果高亮匹配关键词
技术难点
服务端全文检索逻辑（进阶可铺垫后续对接 ES）
客户端防抖输入、实时请求 API
文本高亮处理
